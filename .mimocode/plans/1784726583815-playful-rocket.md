## Implementation Steps

### Step 1: Simplify Autoloader
**File:** `includes/class-autoloader.php`

Replace the complex multi-pattern autoloader with a clean PSR-4 one. Convention: namespace parts become lowercase directories, class names become `class-{name}.php`. The autoloader tries one pattern only — no more guessing with `class-`/`abstract-`/`interface-` prefixes.

```php
spl_autoload_register(function ($class) {
    $prefix = 'AHEcommerce\\';
    $base_dir = AH_ECOMMERCE_DIR . 'includes/';
    if (strncmp($prefix, $class, strlen($prefix)) !== 0) return;
    
    $relative = substr($class, strlen($prefix));
    $parts = explode('\\', $relative);
    $file_name = strtolower(str_replace('_', '-', array_pop($parts)));
    $dir = $base_dir . strtolower(implode('/', $parts)) . '/';
    
    $candidates = [
        $dir . 'class-' . $file_name . '.php',
        $dir . 'abstract-' . $file_name . '.php',
        $dir . 'interface-' . $file_name . '.php',
    ];
    foreach ($candidates as $file) {
        if (file_exists($file)) { require $file; return; }
    }
});
```

### Step 2: Move Abstract_Repository to core/
**Move:** `includes/database/abstract-repository.php` → `includes/core/abstract-repository.php`
**Change namespace:** `AHEcommerce\Database` → `AHEcommerce\Core`
**Add:** `get_paginated()` method to eliminate duplication across repositories.

```php
abstract class Abstract_Repository {
    abstract protected function get_table_name();
    
    public function get($id) { /* existing */ }
    public function insert($data) { /* existing */ }
    public function update($id, $data) { /* existing */ }
    public function delete($id) { /* existing */ }
    
    // NEW: shared paginated query
    public function get_paginated($page = 1, $per_page = 20, $search = '', $search_columns = []) {
        global $wpdb;
        $table = $this->get_table_name();
        $where = '1=1';
        $args = [];
        if (!empty($search) && !empty($search_columns)) {
            $conditions = [];
            foreach ($search_columns as $col) {
                $conditions[] = "{$col} LIKE %s";
                $args[] = '%' . $wpdb->esc_like($search) . '%';
            }
            $where .= ' AND (' . implode(' OR ', $conditions) . ')';
        }
        $offset = ($page - 1) * $per_page;
        $query = "SELECT SQL_CALC_FOUND_ROWS * FROM {$table} WHERE {$where} ORDER BY created_at DESC LIMIT %d OFFSET %d";
        $args[] = $per_page;
        $args[] = $offset;
        $query = $wpdb->prepare($query, ...$args);
        $items = $wpdb->get_results($query);
        $total = (int) $wpdb->get_var("SELECT FOUND_ROWS()");
        return [
            'items' => $items,
            'meta'  => [
                'current_page' => $page,
                'per_page'     => $per_page,
                'total_items'  => $total,
                'total_pages'  => ceil($total / $per_page),
            ]
        ];
    }
}
```

### Step 3: Move Repositories Into Their Modules
**Move** each repository from `includes/database/` into its module directory. Update namespace accordingly.

| Repository | From | To | New Namespace |
|-----------|------|----|---------------|
| `Product_Repository` | `includes/database/class-product-repository.php` | `includes/modules/products/` | `AHEcommerce\Modules\Products` |
| `Order_Repository` | `includes/database/class-order-repository.php` | `includes/modules/orders/` | `AHEcommerce\Modules\Orders` |

Both become simpler — `get_paginated()` is inherited from `Abstract_Repository`, so they only define `get_table_name()` and entity-specific methods (like `Product_Repository::get_meta()`).

# Plugin Admin UI Refactoring Plan

## Context

The CMS plugin admin pages have raw HTML coupled with business logic and data handling. We need to build a fully reusable, modular admin UI framework so new features require zero HTML duplication.

**Already exists:** AdminComponents (10+ methods), admin-style.css (689 lines with design tokens), admin-script.js (268 lines with sortable/media/tabs), AH_Model_Base with CRUD, 87-entry classmap autoloader.

**Built this session (ready to use):**
- `FormBuilder.php` — Fluent form builder with text, textarea, select, checkbox, toggle, radio, editor, image, grid, section
- `TableBuilder.php` — Data table with columns, actions, sortable, empty state
- `PageLayout.php` — Page shell with header, notices, content

**Still needs building:** CardDesigns, AutoSuggester, TabBuilder, expanding all admin pages to use components.

## Execution Phases

### Phase 1: FormBuilder (Core Component)
**File:** `src/Admin/Components/FormBuilder.php`

Create a fluent form builder:
```php
$form = new Ah\Cms\Admin\Components\FormBuilder( 'ah_save_faq', 'ah_faq_nonce' );
$form->text('question', 'Question *', $item->question ?? '', ['required' => true]);
$form->editor('answer', 'Answer *', $item->answer ?? '');
$form->select('status', 'Status', ['active' => 'Active', 'inactive' => 'Inactive'], $item->status ?? 'active');
$form->grid([ [$field1, $field2], [$field3, $field4] ]);
$form->submit('Save FAQ');
$form->render();
```

**Methods:**
- `__construct($nonce_action, $nonce_name)` — wraps form open + nonce
- `text($name, $label, $value, $opts)` — text input
- `textarea($name, $label, $value, $opts)` — textarea
- `editor($name, $label, $value, $opts)` — wp_editor wrapper
- `select($name, $label, $options, $selected, $opts)` — dropdown
- `checkbox($name, $label, $checked, $opts)` — checkbox
- `radio($name, $label, $options, $selected, $opts)` — radio group
- `toggle($name, $label, $value, $opts)` — on/off toggle
- `image($name, $label, $value, $opts)` — WP media picker
- `repeater($name, $label, $rows, $template)` — repeatable field group
- `sortOrder($name, $value)` — sort order with hint
- `dateRange($name, $label, $start, $end)` — day/month/year selects
- `grid($rows)` — 2-column layout for fields
- `section($title)` — section heading divider
- `errors($errors)` — validation error list
- `submit($label)` — submit button
- `render()` — outputs complete form

### Phase 2: TableBuilder (Extract from AdminComponents)
**File:** `src/Admin/Components/TableBuilder.php`

Extract and expand `AdminComponents::dataTable()` into a full builder:
```php
$table = new TableBuilder( $items, ['model' => 'faqs'] );
$table->column('question', 'Question', function($item) { return wp_trim_words($item->question, 12); });
$table->column('status', 'Status', fn($item) => AdminComponents::statusBadge($item->status));
$table->actions(fn($item) => [ 'edit_url' => '...', 'delete_url' => '...' ]);
$table->search('Search FAQs…', $search);
$table->filter('page_id', 'All Pages', $options, $selected);
$table->sortable(true);
$table->emptyState('No FAQs yet. Create your first FAQ.');
$table->render();
```

**Methods:**
- `__construct($items, $opts)` — items array + config
- `column($key, $label, $renderer)` — add column
- `actions($callback)` — row action buttons
- `search($placeholder, $value)` — search input
- `filter($name, $label, $options, $selected)` — dropdown filter
- `bulkActions($actions)` — bulk action select + apply
- `sortable($enabled)` — drag handle column
- `emptyState($message)` — shown when no items
- `render()` — outputs complete table with wrap

### Phase 3: PageLayout (Page Shell)

```php
$page = new PageLayout( 'FAQs', 'dashicons-editor-help' );
$page->notice( $notice, 'success' );
$page->content( $html );  // or $page->render(function() { ... })
$page->render();
```

### Phase 4: Expand AdminComponents
Add missing methods to existing `AdminComponents.php`:
- `tabBar($tabs, $active)` — tab navigation
- `emptyState($message, $icon)` — empty state placeholder
- `confirmDelete($url)` — standardized confirm dialog
- `formSection($title, $fields)` — card with heading + grouped fields
- `mediaField($name, $label, $value)` — WP media picker field
- `stickyHeader($backUrl, $title, $submitLabel)` — edit form header bar
- `statCard($value, $label, $icon)` — dashboard stat card
- `radioCards($name, $options, $selected)` — visual radio card group

### Phase 5: Refactor Admin Pages (One at a Time)
Refactor each admin page to use the new components. **Order by complexity (easy first):**

1. **faqs.php** (170 lines → ~80 lines) — Already uses AdminComponents, upgrade to FormBuilder + TableBuilder
2. **reviews.php** (312 lines → ~120 lines) — Same pattern
3. **audit-log.php** (60 lines → ~30 lines) — Simple read-only table
4. **spotlights.php** (301 lines → ~120 lines) — Add tabBar component
5. **events.php** (308 lines → ~130 lines)
6. **newsletter.php** (313 lines → ~130 lines)
7. **file-links.php** (415 lines → ~180 lines)
8. **reference-notes.php** (371 lines → ~160 lines)
9. **taxonomy.php** (429 lines → ~180 lines)
10. **notices.php** (582 lines → ~250 lines) — Complex, has radio cards
11. **navigation.php** (712 lines → ~300 lines) — Most complex builder UI
12. **posts.php** (1275 lines → ~500 lines) — Largest, most complex

**For each page:**
1. Read current file
2. Identify business logic vs rendering
3. Extract POST handling into controller method (keep inline for now, move to controller later)
4. Replace raw HTML with FormBuilder/TableBuilder/PageLayout calls
5. Run `php -l` on modified file
6. Verify page still works in browser

### Phase 6: Controller Extraction (Optional — after all pages use components)
For the largest pages, extract POST handling into feature controllers:
- `src/Feature/Faqs/Controller/FaqsAdminController.php`
- `src/Feature/Reviews/Controller/ReviewsAdminController.php`
- etc.

**This is optional** — the component refactoring alone eliminates HTML duplication. Controller extraction is a further cleanup.

## Verification

After each page refactor:
1. `php -l` on modified files
2. Visit admin page in browser — verify list/form views work
3. Test create/edit/delete operations
4. Verify CSS styling matches before refactoring
5. Check no JS errors in browser console

- `AH_Model_Base` — CRUD pattern works, keep as-is
- `AH_Admin_Menus` — routing works, keep as-is
- `AH_Admin_Bootstrap` — hook registration works, keep as-is
- Any page's business logic (model calls, nonce verification, sanitization)

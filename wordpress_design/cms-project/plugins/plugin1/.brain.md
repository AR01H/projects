## Architecture Decisions

### Table Prefix Convention
- All custom tables: `$wpdb->prefix . 'ah_' . $name` (e.g. `wp_ah_pages`)
- Never hardcode `wp_` — always `$wpdb->prefix`
- Helper: `AH_DB_Helper::table('pages')` returns the full table name

### Class Loading
- `inc/class-autoloader.php` — static class map, no PSR-4 directory scan
- All new classes MUST be added to the map in `AH_Autoloader::$map`
- Pattern: `'AH_ClassName' => 'relative/path/to/class-file.php'`

### Admin Page Routing
- URL param `page=ah-{slug}` routes to admin submenu
- URL param `tab=...` switches sub-tabs (sanitize_key)
- URL param `action=edit&id=N` loads edit mode
- POST with nonce = mutation; GET with _wpnonce = delete/action links
- Nonce action pattern: `ah_save_{section}`, `ah_del_{section}_{item}`

### AJAX Pattern
- All AJAX actions registered in `AH_Ajax_Handlers::init()`
- JS sends via `ahAjax({ action, nonce, ...data })`
- Server: `self::verify()` checks capability + nonce (`ah_admin_nonce`)
- Allowed tables/models are whitelisted arrays — never trust raw POST

### Model Pattern
- Abstract base: `models/class-model-base.php` (AH_Model_Base)
- All CRUD auto-logs to `wp_ah_audit_logs` via `AH_DB_Helper::log_action()`
- Models do NOT echo — they return data or bool
- Image URLs resolved via `AH_Media_Model::get_url(id)` (never stored raw)

### Plugin Mode (Dual Deployment)
- `ah-cms.php` — WordPress plugin main file; defines `AH_PLUGIN_DIR/URL` with `untrailingslashit()`, loads autoloader, inits admin + AJAX + shortcodes, registers activation hook for DB install
- `functions.php` — checks `defined('AH_PLUGIN_DIR')` first; if plugin active, only adds theme supports + `AH_Asset_Loader::init()` then returns early (no constants, no admin bootstrap)
- `AH_Asset_Loader` — always uses `get_template_directory_uri()` (not `AH_THEME_URL`) so CSS/JS resolves to the ACTIVE THEME folder regardless of deployment mode
- Backward-compat aliases: `AH_THEME_DIR = AH_PLUGIN_DIR`, `AH_THEME_URL = AH_PLUGIN_URL` — all 60+ existing files work unchanged in plugin mode
- Plugin deploy: rename folder to `ah-cms`, move to `wp-content/plugins/`, activate in WP admin
- Companion theme's `functions.php` only needs: `add_action('after_setup_theme', ...)` for theme supports + `AH_Asset_Loader::init()`

### Security Standards
- All DB queries: `$wpdb->prepare()` — no exceptions
- All output: `esc_html()`, `esc_attr()`, `esc_url()`, `esc_textarea()`
- All POST input: sanitize before use (`sanitize_text_field`, `sanitize_textarea_field`, `esc_url_raw`, `(int)`, `sanitize_key`)
- Nonce verification before any mutation
- `manage_options` capability check at top of every admin page

## JS Patterns (admin-script.js)

| Selector | Behavior |
|---|---|
| `.ah-sortable-list` | jQuery UI Sortable → `ah_update_sort_order` on stop |
| `[data-toggle-status]` | AJAX toggle, swaps badge color + button text |
| `[data-delete-item]` | confirm() → AJAX delete → row fade out |
| `.ah-pick-image` | Opens WP Media Library, populates `.ah-image-id` + `.ah-image-preview` |
| `.ah-remove-image` | Clears image ID and preview src |
| `.ah-add-repeater` | Clones last `.ah-repeater-item` with blank inputs |
| `.ah-repeater-remove` | Removes parent `.ah-repeater-item` (min 1 preserved) |
| `.ah-generate-slug-source` | On blur → sanitizes to slug → writes to `.ah-slug-field` |
| `.ah-tab[data-tab]` | Client-side tab switching via history.replaceState |
| `#ah-bulk-select-all` | Checks/unchecks all `.ah-bulk-cb` checkboxes |

## Naming Conventions

| Thing | Convention | Example |
|---|---|---|
| PHP Classes | PascalCase with AH_ prefix | `AH_Home_Model` |
| PHP Methods | snake_case | `get_page_header()` |
| PHP Files | kebab-case with class- prefix | `class-home-model.php` |
| DB tables | snake_case with ah_ prefix | `wp_ah_home_why_us_cards` |
| Nonce actions | snake_case with ah_ prefix | `ah_save_home`, `ah_del_service` |
| AJAX actions | snake_case with ah_ prefix | `ah_toggle_status` |
| CSS classes | BEM-lite with ah- prefix | `.ah-btn`, `.ah-btn-primary`, `.ah-card-header` |
| JS objects | camelCase | `ahAdmin.ajaxUrl` |
| Admin page slugs | kebab with ah- prefix | `ah-services`, `ah-home-sections` |
